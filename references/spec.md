# 完整数值规范与构建细节 v3

> 配合 SKILL.md 使用。安全区坐标来自官方《设计规范》图（权威值）；这里是精确数值、配色、节点构建顺序与踩坑记录。

## 一、尺寸与安全区坐标（权威）

### 活动中心 Banner 1035×368
| 元素 | 规格 |
|---|---|
| 画布 | 1035 × 368 |
| **红色固定边距（禁文字）** | 上/左/右 ≈ 40，下 ≈ 94 |
| **左侧文字灰框（唯一可放文字区）** | **x ∈ [45, 580]、y ∈ [41, 274]**；可用宽 ≤535 |
| 文字对齐/定位 | 左对齐（左边距 ≈47）；**文案块整体在灰框内垂直居中**（`y = 41 + (233 − 块高)/2`） |
| 标签 | ≥30px（实测 ~27–30）；圆角底徽章可沿用源稿 |
| 主标题 | **≥45px**（实测 46–54），Heavy；行高 110–120% |
| 副标题 | **≥36px 且最多一行**（实测 30–36），Medium；行高 130% |
| **右侧主视觉安全区** | **x ∈ [625, 994]、y ∈ [41, 274]**；视觉可超框 bleed，核心落此框 |
| 合规 | **无**（Banner 永不加） |

> 文案块过宽/过高时：把该帧所有文字**编组→`rescale(factor)` 等比微缩**（factor=min(1, 531/块宽, 229/块高)），再左对齐 x47、垂直居中。**注意用增量法定位，见"坐标铁律"。**

## 二、视觉三范式（见 SKILL §3.5）
- **A 主体/产品型** → `rescale(帧高/源高)`，满高、右对齐、不裁切。
- **B 满幅场景型** → 覆盖式缩放（宽≥帧宽）、超框、`clipsContent` 帧裁剪；文字叠留白侧，长文案缩字号。
- **C 居中构图型** → 收进右侧 ~400 宽区、垂直居中、不侵入文案。

## 三、坐标铁律（本轮两次返工根因）
- **帧内节点定位用相对父级/增量法**：`node.x += 目标abs − node.absoluteBoundingBox.x`（y 同）。**绝不把绝对坐标直接赋给 `.x/.y`** —— 会翻倍/跑框/空白渲染。
- **CTA 作为弹窗帧的直接子节点新建**（坐标简单 0–900），用自动布局 hug + padding72；比在嵌套框里改坐标可靠。若旧按钮嵌在自动布局容器里，直接**删旧建新**（采集原 fills/文案/字体/色后重建）。
- 重排大字号文本前把 `lineHeight` 设 `{unit:'AUTO'}`，否则被小的固定行高压/裁。
- 合规若在被删按钮的同一容器里，删按钮后会被自动布局上移 → 把合规**reparent 到弹窗帧**、`y=1150`、居中。

## 四、常用配色（示例，随活动主题）
- 老虎黄 CTA：底 #252933，字 #FFEB33。棕金：金渐变，字深棕 #392318，文案奶白 #FEF4E7。
- 复用源稿渐变/图片填充：读源 `fills` 直接套到新元素，保证配色一致（图片填充的 imageHash 在同文件内可复用）。

## 五、构建顺序（use_figma，每步≤10操作并截图自检）
1. 建目标 frame，`clipsContent=true`；弹窗 `cornerRadius=20`。
2. 克隆源背景 → 铺满。
3. 克隆主视觉 → 按视觉三范式定位（A `rescale`满高右置 / B 覆盖裁剪 / C 收右区）。**不裁切=等比整块。**
4. 文案：Banner 放左灰框 [45,580]×[41,274]、垂直居中；弹窗放安全区、居中。字体按 §2、行高分层。
5. 弹窗 CTA：**作为帧直接子节点**新建自动布局胶囊（fills 沿用主题），文字**字号60 / 字重 Medium**，120高/圆角60，`y=1010`。**宽度**：量文字宽 tw —— tw≤244 则 `primaryAxisSizingMode='FIXED'`+`resize(384,120)`+`primaryAxisAlignItems='CENTER'`（固定384）；tw>244 则 `primaryAxisSizingMode='AUTO'`+`padding72`（hug=tw+144）。最后 `x=(900−W)/2`。
6. 弹窗合规：14px/70%、居中、`y=1150`。Banner 不加合规。
7. 克隆帧 → 繁/英：仅改文案+切字体；CTA hug 自动重算宽、重新居中。逐帧截图确认文字渲染、不跑框。

## 六、字体落地
- **默认直出**：MiSans / MiSans TC / Roboto（已上传 Figma 账号，云端可 `loadFontAsync`）；思源 CN/HK 为后备。字重：主 Heavy、副 Medium、正文/合规 Regular、标签 Bold、**CTA Medium**。
- **按需苹方**：`references/font-swap/` 的 Tiger FontSwap 本地插件（本机桌面版导入 manifest；金属字不动）。

## 七、踩坑记录
- **坐标翻倍/空白渲染**：绝对坐标赋给帧内节点导致（见坐标铁律）。用增量法根治。
- **CTA 嵌套框改不动**：容器自动布局覆盖子坐标 → 删旧建新为帧直接子节点。
- **大字号被压**：固定小行高 → 设 AUTO。
- **改文案先解决字体**：编辑前 `loadFontAsync` 当前字体；缺失可直接把 `fontName` 设成已加载目标字体（替换不需加载旧字体）。
- **英文易超长**：主标题超宽降字号到单行；副标题缩到 ≤1 行不进视觉区。
- **截图同步延迟**：`use_figma` 数据改动后 `get_screenshot` 可能返回 1×1/滞后帧；数据层已改则以数据核对为准，等几秒再截图。
