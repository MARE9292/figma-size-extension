# 完整数值规范与构建细节

> 配合 SKILL.md 使用。这里是精确数值、配色、节点构建顺序与踩坑记录。

## 一、尺寸与坐标

### 活动中心 Banner 1035×368
| 元素 | 规格 |
|---|---|
| 画布 | 1035 × 368 |
| 左侧文案左边距 | x ≈ 57（固定边距） |
| **文案顶部** | **y = 105（主标题顶部，硬规则）** |
| 主标题 | ≥45px，加粗(Heavy)，y=105 |
| 副标题 | ≥36px，Medium，单行，主标题下方 ~80px 间距（y≈185） |
| 标签/角标 | ≥30px；顶部角标徽章可沿用源稿（圆角底） |
| 右侧主视觉 | **固定在右侧 410px 宽区域 x∈[625,1035]**；主视觉等比缩放至宽≤410，整组放入并垂直居中；不越界、不裁切、不压文字 |
| 左侧文案区 | x∈[0,625]（左边距 57）；主/副标题 right 必须 < 625 |
| 合规 | **无**（Banner 永不加） |

> 实现：克隆主视觉组后用 `node.rescale(factor)` 等比缩放（`resize` 不会缩放 frame 子节点，会导致裁切）。factor = 410 / 原组宽；定位 x=625、y=(368−缩放后高)/2。

### 弹窗 Popup 900×1200
| 元素 | 规格 |
|---|---|
| 画布 | 900 × 1200，圆角 20，导出 PNG |
| 文字水平 | 主/副/合规/按钮 **居中**（文本框宽 = 900−2×60 = 780，x=60，textAlignHorizontal=CENTER） |
| 主标题 | 居中；中文 ~72px Heavy，英文若超宽降到 ~54px 单行 |
| 副标题 | 居中；~40px Medium |
| 核心视觉安全区 | 照片等比缩放居中（高 ~520–560），不裁切，y 从 ~370 起 |
| CTA 按钮 | 高 **120**、圆角 **60**、字号 **60**、文案 hug + 左右内边距 **72**；按钮整体水平居中，y≈980 |
| 合规 | 14px，居中，置底 y≈1140，宽 780，透明度按源稿（约 50%） |

## 二、常用配色（示例，随活动主题调整）
- 老虎黄主题 CTA：底 #252933（r0.145 g0.16 b0.20），字 #FFEB33（r1 g0.92 b0.2）。
- 棕金主题：CTA 金色渐变（226,180,129 → 255,222,186），字深棕 #392318（r0.224 g0.137 b0.094）；文案奶白 #FEF4E7（r0.996 g0.957 b0.906）。
- 复用源稿渐变：读源节点 `fills` 里的 GRADIENT_LINEAR paint，直接套到新元素，保证配色一致。

## 三、构建顺序（use_figma，每步≤10操作并截图自检）
1. 建目标尺寸 frame，`fills` 复用源帧纹理填充；`clipsContent=true`；弹窗设 `cornerRadius=20`。
2. 克隆源背景渐变节点 → resize 铺满。
3. 克隆主视觉照片节点 → 等比缩放（`resize(w*scale,h*scale)`，scale 由目标高/原高算）→ 定位（banner 贴右 / 弹窗居中）。**不裁切 = 等比缩放整块。**
4. 克隆/复用角标徽章；其文字重置为目标语言+字体。
5. 新建主标题/副标题（思源/Roboto，按字重与字号）；按版面不变量定位。
6. 弹窗：新建 CTA 胶囊（金/深底渐变填充 + 文字），按 120/60/60/72 规格、居中。
7. Banner：**不要**加合规。弹窗：加居中合规 14px。
8. 克隆该帧 → 繁/英：仅改文案 + 切字体（繁 Source Han Sans HK / 英 Roboto）；CTA 文案变化后**重算按钮宽=文字宽+144 并重新居中**。

## 三点五、文字排版（所有物料通用，硬规则）
- 主标题、副标题 **行高 = 130%**（`lineHeight = {unit:'PERCENT', value:130}`）。
- **主标题与副标题之间垂直间距 = 20px**（设 `副标题.y = 主标题.y + 主标题.height + 20`，在行高已设为 130% 之后计算）。
- **3 行例外**：当文案是 3 个单独上下排列的文本元素（主标题/产品字/副标题），三者行间距 = **0px**（VERTICAL 自动布局 `itemSpacing=0`）。仅 2 个文本且其中某个自身折行 ≠ 3 行，仍 20px。
  - 此 3 元素情况下，**Banner 文案整体底对齐：文字框 bottom 距帧底 110px**（即 `tf.y = 368 − 110 − tf.height`），覆盖 top=105。
- **弹窗合规距 CTA = 40px**（所有语言）：`合规.y = CTA.y + CTA.height(120) + 40`。
- **元规则·设计稿排版优先**：需求写明"按设计稿标题排版样式出"时，跳过本规范文字排版/位置规则，直接复刻设计稿排版出图。
- Banner 中主标题 top 仍恒 = 105px；副标题随之 = 105 + 主标题高 + 20。

## 四、字体落地

### 默认：思源（直出，无需插件）
`Source Han Sans CN` / `Source Han Sans HK` 在云端 `use_figma` 可直接 loadFontAsync。字重映射：主标题 Heavy、副标题 Medium、正文/合规 Regular、标签/按钮 Bold。

### 按需：苹方 / MiSans（本地插件）
云端环境**加载不了苹方/MiSans**（插件 API 禁止给未加载字体赋值；非企业版无法上传共享字体）。只能在**本机 Figma 桌面版**落地：用 `references/font-swap/` 里的 **Tiger FontSwap** 插件。
- 一次性：本机装好字体（苹方 macOS 自带；MiSans 自行安装）→ Figma 桌面版 → 插件 → 开发 → Import plugin from manifest → 选 `manifest.json`。
- 用法：插件面板点「苹方 / MiSans」按钮，或把 `AUTORUN_MODE` 写成 `'PingFang'`/`'MiSans'` 后运行即自动替换。
- 它按帧名映射语言，遇新文件需在 `code.js` 的 `FRAME_LANG` 增加帧名。
- 金属/产品字（如 Roboto Condensed wordmark）始终不动。

## 五、踩坑记录
- **云端 MCP 与桌面字体隔离**：`use_figma` 在云端渲染，看不到本机字体；本地桌面 Dev MCP 是只读的。所以苹方/MiSans 必须靠本地插件落地。
- **改文案先解决字体**：编辑/对齐文本前必须 `loadFontAsync` 该节点当前字体；若当前字体缺失，可直接把 `fontName` 设成已加载的目标字体（替换不需要加载旧字体）。
- **克隆后 CTA 居中**：换语言后文字宽变了，务必按 `按钮宽=文字宽+144` 重算并 `x=(900-按钮宽)/2`。
- **英文易超长**：主标题超宽就降字号到单行；副标题缩短到 ≤1 行且不进主视觉区。
- **同步延迟**：本地插件改完，云端 MCP 读取需等几秒同步；先读探针(如 sharedPluginData lastlog)再读结果。
