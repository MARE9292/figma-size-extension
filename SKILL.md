# Figma 尺寸延展规范 v3（Tiger Size Extension）

把**一张源设计稿**延展成标准投放尺寸 × 三语言，保证团队产出**规范一致**。这是硬规范，按它执行，不要自由发挥版面。v3 的安全区坐标来自官方《设计规范》图（红=禁文字固定边距，灰=文字/视觉安全区），为权威值。

## 0. 触发后先做
1. 读源稿：`get_metadata` → `get_design_context` → 截图，看清内容、字号、图层、图片填充（记下 imageHash 以便复用）。
2. 判定场景：源稿属于「活动中心 Banner」类（宽幅 文字+主视觉）还是「弹窗」类（竖版促销）；并判定主视觉属于**视觉三范式**里的哪一种（见 §3.5）。
3. 跟用户确认：目标尺寸、语言、英繁文案是否由你翻译（默认是，交付前请市场复核）。
4. 严格执行下面的安全区与版面不变量。逐帧用截图自检。

## 1. 标准尺寸 × 语言
| 场景 | 尺寸(px) | 语言 |
|---|---|---|
| 活动中心 Banner | **1035 × 368**（宽） | 简 / 繁 / 英 |
| 弹窗 Popup | **900 × 1200**（竖） | 简 / 繁 / 英 |

命名：`<活动名>-活动中心-1035x368-{简略空/繁/EN}`、`<活动名>-弹窗-900x1200-{/繁/EN}`。

## 2. 字体（默认）
- **简体中文 → MiSans**；**繁体 → MiSans TC**；**英文 → Roboto**。（字体已上传 Figma 账号，云端 `use_figma` 可直接 `loadFontAsync`。）
- **后备**：MiSans 不可用时用思源 **Source Han Sans CN / HK**。
- 字重：主标题 **Heavy**（英 Bold/ExtraBold）、副标题 **Medium**、正文/合规 **Regular**、标签 **Bold**。
- **弹窗 CTA 按钮文字字重 = Medium（硬规则）**。
- **行高**：展示大标题（主标题 / 金属产品字）**110–120%**；副标题 / 正文 **130%**。
- **主副间距** 20px（副标题 top = 主标题 bottom + 20）。
- **3 行例外**：文案为 3 个单独上下排列文本元素（主标题/产品字/副标题）时，三者行间距 = **0px**（`itemSpacing=0`）。仅 2 文本其中自身折行不算 3 行，仍 20px。
- 金属/产品字（如 iPhone wordmark = Roboto Condensed Black Italic）**永不替换**。
- **设计稿排版优先（元规则）**：需求写明"按设计稿标题排版样式出"时，跳过本规范文字排版，直接复刻源稿排版。
- 仅当用户**明确要求苹方**时，才用 `references/font-swap/` 本地插件落地。

## 3. 安全区与版面不变量（硬规则，坐标为权威值）

### 活动中心 Banner 1035×368
- **红色固定边距 = 禁止放文字**（上/左/右≈40、下≈94）。
- **文字只能在左侧灰框内：x ∈ [45, 580]、y ∈ [41, 274]**（可用宽 ≤535）。左对齐（左边距 ≈47）；**文案块整体在灰框内垂直居中**（不再是"恒 top=105"）。
- 标签 ≥30px；**主标题 ≥45px**（实测 46–54）；**副标题 ≥36px 且最多一行**（实测 30–36）。
- **右侧主视觉安全区 = 右灰框 x ∈ [625, 994]、y ∈ [41, 274]**；视觉可超框 bleed，但**核心元素落在此框内**；文左·视觉右。
- **永远不要加合规文字。**

### 弹窗 Popup 900×1200
- 容器圆角 **20px**、导出 **PNG**；**广告角标系统自带、不手工加**。
- 结构（上→下）+ 安全区：
  - **主标题/副标题安全区**：x ∈ [55, 844]、y ∈ [95, 330]。
  - **核心视觉安全区**：x ∈ [25, 876]、y ∈ [350, 950]（视觉可超框，核心落此框）。
  - **CTA 行动按钮**：y ∈ [1010, 1130]。
  - **合规**：置底，CTA 下方（y ≈ 1150）。
- **CTA 按钮（永远要有，源稿没有也补）**：胶囊形，**高 120、圆角 60、字号 60、字重 Medium、水平居中**（x=(900−W)/2）；配色随活动主题（黑底黄字 / 金底深字 / 图片渐变填充等，沿用源稿）。
  - **宽度规则**：文字宽 **≤244px → 按钮固定 384px（最小写死值）**，文字在其中居中；文字宽 **>244px → 按钮宽 = 文字实际宽 + 左右各 72（即文字宽+144）**、hug。即 `W = 文字宽>244 ? 文字宽+144 : 384`。
- **合规**：**14px、透明度 70%、水平居中**、宽约 780、位于 **CTA 底 + ≈20px**。
- 主/副标题默认水平居中；**左右分栏构图（如转仓）可左对齐，跟随源稿**。
- 主视觉整体等比缩放、不裁切，核心落视觉安全区。

## 3.5 视觉三范式（决定跨尺寸适配手法 —— 通用度关键）
| 范式 | 特征 | 适配法 |
|---|---|---|
| **A 主体/产品型**（人物照、产品罐、卡组） | 独立主体 | 满高铺满帧高、右对齐、宽随高等比、**完整不裁切**（factor=帧高/源高） |
| **B 满幅场景型**（Apple Pay 手机场景） | 场景即背景 | 覆盖式（宽 ≥ 帧宽甚至更大）、超框 bleed、**帧裁剪**；文字叠在留白侧；长文案缩字号避让主体 |
| **C 居中构图型**（奖杯等横向主体） | 主体居中、横向宽 | 收进**右侧约 400 宽区**、垂直居中、**不侵入文案区**（满高会横侵，故用此例外） |
> 共性：视觉一律做**完整大图 + 超框 bleed + 帧裁剪**；换尺寸只重定位/缩放，**不重裁图**。核心落公共安全区，bleed 区只放可裁装饰。

## 3.8 图层与视觉处理（硬规则）
- **图层不合并**：保留分层结构，克隆源视觉图层复用，禁止拍平成 PNG 当底图。
- **视觉不裁切**：视觉保持完整，可超出画板，靠帧 `clipsContent` 显示。
- **文字置顶**：文字图层永远在视觉之上。

## 3.9 坐标铁律（本轮返工根因，务必遵守）
- **给"帧内节点"定位必须用相对父级坐标或增量法**（`node.x += 目标abs − node.absoluteBoundingBox.x`），**绝不能把绝对坐标直接赋给 `.x/.y`** —— 帧内节点的 `.x` 是相对父级的，直接赋绝对值会把节点推到 2× 位置、跑出画板被裁 → 空白渲染。
- **CTA 用自动布局实现**：`layoutMode='HORIZONTAL'` + `primaryAxisSizingMode='AUTO'`（hug）+ `paddingLeft/Right=72`，宽度自适应各语言文案；把 CTA 作为**弹窗帧的直接子节点**创建（坐标简单 0–900），规避嵌套框/自动布局的坐标陷阱。
- 文本行高被设成很小的固定 px 会压/裁大字号，重排前把 CTA/标题的 `lineHeight` 设为 `{unit:'AUTO'}`。

## 4. 高效做法
- 以源稿为母版：**克隆**源节点复用主视觉、产品金属字等（不重做）。
- 先做「某尺寸+简体」调好，再**克隆出繁/英**，仅改文案+切字体，三语言版面一致。
- 只做尺寸/排版适配，**不改视觉风格、不越安全区**。
- 文案过长（多见英文）：缩短副标题保证 ≤1 行；或整体等比微缩使文案块落进灰框。

## 5. 自检清单（交付前逐项核对）
- [ ] 尺寸正确（1035×368 / 900×1200），语言齐全
- [ ] 字体：简 MiSans / 繁 MiSans TC / 英 Roboto（后备思源），字重对；**弹窗 CTA 文字 Medium**
- [ ] Banner：文字全部在**左灰框 [45,580]×[41,274]** 内、红边距零文字、文案块垂直居中；副标题单行；无合规；视觉未裁切、核心在右灰框
- [ ] 弹窗：圆角20；CTA 高120/圆角60/字号60/文字 Medium、居中；**CTA 宽=文字宽>244?文字宽+144:384（最小384）**；合规 14px/**70%**、居中、CTA 下方 ≈20px
- [ ] 坐标用增量法/相对坐标；逐帧截图确认文字渲染正常、无裁切/重叠/跑框
